01 · Project Overview
What We're Building

G&G Wealth Management is a personal budgeting app designed to help paycheck-to-paycheck clients build financial confidence through simple, visual budget tracking — with direct coach oversight from Gustavo & Greys.

The Problem It Solves

Most budgeting apps are built for people who already have good financial habits. G&G is built for people who are starting from zero — living paycheck to paycheck — and need a system that builds confidence gradually, with a real human coach guiding them every step of the way.

Two-Sided App

The app has two distinct sides:

  • Client App (Mobile) — what clients use daily to log expenses, track debts, save money, and message their coach
  • Coach Dashboard (Web/Desktop) — what Gustavo & Greys use to monitor all clients, send coaching notes, and export PDF reports
Target Clients

Paycheck-to-paycheck individuals who want to break the cycle, build an emergency fund, eliminate debt using the Debt Snowball Method, and develop lasting financial discipline — with a coach in their corner.

03 · App Architecture
Budget Categories (The 4 Rings)

The core of the app is a 4-ring budget system inspired by fitness tracker rings. From outermost to innermost:

  1. Debt Payoff (Dark Red #8B1A1A) — outermost, largest ring
  2. Savings (Dark Green #1A5C2E) — second ring
  3. Wants (Sand #C8B48A) — third ring
  4. Fixed (Muted Orange #C97A2A) — innermost, smallest ring

⚠️ Important naming: What most apps call "Needs" is called "Fixed" in this app — representing fixed monthly expenses like mortgage, car payment, phone bill, etc.

Budget Styles (Onboarding)

Clients choose a budget style during onboarding with their coach:

  • Aggressive Saver — maximum toward debt and savings
  • Balanced — even split across categories
  • Gentle Start — easier entry point for beginners
  • Custom — fully adjustable sliders per category
Income Model
  • Income is entered manually each week — clients enter their paycheck amount
  • Budget rings recalculate automatically based on income and chosen budget style
  • Monthly budget is used for Fixed expenses only (not weekly)
Fixed Expenses Logic
  • Clients enter fixed monthly bills once (mortgage, electric, phone, car, etc.)
  • Bills persist automatically — no re-entry needed each month
  • Clients check off each bill as paid
  • When checked off, the bill auto-posts to the Daily Log for that day
  • Unchecking removes it from the log
Debt Snowball Method
  • Clients enter each debt individually with name, type, total balance, amount already paid, and minimum monthly payment
  • Debts are auto-sorted smallest to largest remaining balance
  • First unpaid debt is marked as the 🎯 Target
  • When a debt hits zero — subtle gold shimmer celebration screen appears
  • Clients can add, edit, or delete debts at any time
Streak System

Two simultaneous streaks are tracked and displayed:

  • 🔥 Days Logged — consecutive days the client logged at least one expense
  • ✅ Days On Track — consecutive days the client stayed under budget in all categories

05 · Backend Development Requirements

The prototype is fully functional as a UI demo. To make it a real working app used by real clients, the following backend infrastructure is needed:

Authentication
  • Real username & password login system
  • Separate client and coach account types
  • Password reset / forgot password flow
  • Session management — stay logged in
  • Recommended: Firebase Authentication
Database
  • Store all client data — expenses, income, debts, savings goals, fixed bills
  • Store all coach notes and client messages
  • Real-time sync — client logs expense, coach dashboard updates instantly
  • Recommended: Firebase Firestore (real-time) or Supabase
Data Structure (per client)
clients/{clientId} ├── profile: { name, email, coachId, createdAt } ├── budget: { weeklyIncome, style, ringPercentages } ├── expenses/{expenseId}: { amount, category, desc, timestamp } ├── fixedExpenses/{id}: { name, amount, due, paid, paidDate } ├── debts/{debtId}: { name, type, balance, paid, min, paidOff } ├── savings/{goalId}: { name, goal, current, weekly, complete } ├── emergencyFund: { current, goal } ├── streaks: { logged, onTrack, lastLogDate } └── messages/{msgId}: { from, text, timestamp }
Real-Time Coach Dashboard
  • Coach dashboard listens to all client data in real time
  • When a client logs an expense — it appears on coach dashboard instantly
  • Unread message dots update in real time
  • Client status (On Track / At Risk / Over Budget) recalculates automatically
  • Coach can see which clients are currently active in the app
Messaging System
  • Real-time two-way messaging between coach and each client
  • Push notification when coach sends a note (future feature)
  • Message history stored permanently per client
PDF Reports
  • Currently works via browser print — functional for now
  • Future upgrade: server-side PDF generation (Puppeteer or similar) for one-click download
Hosting & Deployment
  • Client App: Wrap as iOS & Android app using React Native or Capacitor
  • Coach Dashboard: Deploy as web app — Firebase Hosting or Vercel
  • App Store: Submit to Apple App Store and Google Play Store

💡 Recommended Tech Stack: React Native (mobile) + Firebase (auth + database + hosting). This is the fastest path to a production app using the prototype as a complete design reference. All screens, colors, logic, and features are already decided — the developer only needs to wire up the backend.

08 · Instructions for Developer
Step 1 — Review the Prototype

Open gg-overview.html in a browser. This shows all 9 screens with descriptions and live links to each one. Click through every screen to understand the full user experience before writing any code.

Step 2 — Understand the Two Sides
  • Client side (gg-login through gg-weekly-summary) — mobile app, max width 390px
  • Coach side (gg-coach-dashboard) — web/desktop interface, max width 480px currently but can expand
Step 3 — Recommended Tech Stack
  • Frontend: Convert HTML/CSS/JS to React Native for mobile, React for coach web dashboard
  • Backend: Firebase (Authentication + Firestore + Hosting)
  • PDF Generation: Puppeteer or React-PDF for server-side PDF export
  • App Packaging: Expo (React Native) for iOS + Android
Step 4 — Key Business Rules to Preserve
  • Fixed expenses only appear in the Daily Log after being checked off as paid
  • Debts must always sort smallest to largest remaining balance — never manually reordered
  • Coach dashboard client list must always sort: Over Budget → At Risk → On Track
  • Clients are completely private from each other — no cross-visibility
  • The word "Needs" is never used — it is always "Fixed" in this app
  • The word "Advisor" is never used — it is always "Coach" in this app
  • Budget score is 0-100 — calculate based on % of categories on track
Step 5 — Design Rules to Preserve
  • Default mode is dark — black background throughout
  • Primary font is Inter for all UI, Alex Brush for brand/script only
  • Gold sparkle animation runs on every screen background
  • All primary action buttons use the gold shimmer animation
  • Celebrations are subtle — gold shimmer only, no confetti or loud animations
  • Over-budget warnings use red (#E05555) with a pulsing animation

📌 To the developer: This prototype was built screen by screen with the client. Every color, font, label, icon, and interaction was deliberately chosen. Please preserve all design decisions exactly as built. If you have questions about any specific decision, refer to this document or the prototype files directly.
